# XENO Browser App

Firebaseを使った、XENOカードゲームのオンライン対戦ブラウザ実装です。

## 起動方法

1. `index.html` をブラウザで開く
2. そのままプレイ可能

## Firebase設定

1. Firebaseプロジェクトを作成
2. Authenticationで`匿名`を有効化
3. Firestore Databaseを作成
4. `firebase-config.example.js` を `firebase-config.js` にコピーして値を設定

## ユーザー作成とログイン

1. トップ画面でユーザー名を入力して `作成`
2. `作成済みユーザー` の一覧から選択して `ログイン`

注記:

- 内部的には Firebase Auth の匿名認証を使用します。
- 作成したユーザー名は Firestore の `users_public` に保存され、ログイン時に一覧表示されます。

## ルームIDで2人対戦

1. 2つのタブまたは2つのウィンドウで `index.html` を開く
2. 両方でログイン
3. 片方でルームID（`1`〜`9` の1桁）を入力し `ルーム作成`
4. もう片方で同じルームIDを入力し `ルーム参加`
5. ルーム作成側は待機画面へ遷移し、マッチ成立後に対戦画面へ自動遷移
6. 対戦終了時は勝者ユーザー名を表示してトップ画面に戻る

注意:

- ルーム機能は Firestore を使って同期します。Firebaseルールのデプロイが必要です。

## GitHub公開

1. このリポジトリをGitHubにpush
2. `firebase-config.js` を実値で作成してコミット
3. GitHub PagesかFirebase Hostingで配信

## 画面構成

- トップ画面: ユーザー作成、ログイン、ルーム作成/参加
- 待機画面: マッチング待ち
- 対戦画面: 左側に相手の動き、右側に自分の手札
- カード下部に効果テキストを表示

## ルール実装について

- カード構成は公式準拠です（全18枚: 1〜8が各2枚、9・10が各1枚）。
- 英雄(10)の「自分からは捨てられない」制約を実装しています。
- 転生札1枚を伏せる処理と、死神での英雄捨て時の復活を実装しています。
- 皇帝と革命(少年2枚目)の公開処刑を実装しています。
- 対戦形式はルームID方式の2人対戦のみです。

## 構成

- `index.html`: 画面の骨組み
- `styles/main.css`: スタイル
- `src/cards.js`: カード定義と山札生成
- `src/game.js`: ゲームロジック本体
- `src/ui.js`: DOM描画
- `src/main.js`: エントリポイント
